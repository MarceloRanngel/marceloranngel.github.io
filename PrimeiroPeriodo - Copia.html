Segundo Período


Abaixo você pode encontrar todos os tópicos que foram abordados no código, além de encontrar os links contendo o código em PDF e GitHub.

Tema do Projeto :

Fazer um programa em Java, com interface gráfica e armazenamento dos dados em banco de dados (SGBD PostgreSQL), que permita fazer o controle de vendas de produtos em uma loja fictícia.

Programação Orientada a Objetos

O programa deverá ter, pelo menos, as seguintes funcionalidades:

Cadastro de Clientes: fazer a manutenção dos dados dos clientes. Para cada cliente, deverão ser registrados os seguintes campos: CPF (campo chave), nome completo, endereço (rua, complemento, setor, cidade, UF e CEP) e os telefones do cliente (para cada telefone: ddd, número e tipo – celular, residencial, comercial, ...);

Cadastro de Produtos: código do produto (EAN-13, campo chave), descrição, preço unitário e a unidade na qual o produto é vendido (gramas, metros, unidade, ...);

Registro de Vendas: número da nota fiscal (campo chave), data da venda, cliente, valor desconto, valor total da compra, forma de pagamento (cartão de crédito, cartão de débito, em dinheiro ou no boleto) e a lista dos produtos comprados (para cada item da nota: produto, quantidade, valor unitário).

Consultas (em tela) realizadas por período (usuário deverá fornecer data inicial e data final):
- Vendas por produto;
- Vendas por cliente;
- Nota fiscais emitidas.

Exportação dos dados para um ou mais arquivos texto no formato CSV (campos separados por pontoe-vírgula);

Importação dos dados a partir de um ou mais arquivos texto no formato CSV;

Relatórios estatísticos definidos no componente curricular “Estatística Aplicada”.

Professor(es) responsável(is) : Willians Santos

Projeto de Banco de Dados

Fazer a modelagem do banco de dados que será usado para armazenar os dados do programa descrito no componente curricular “Programação Orientada a Objetos”.

Professor(es) responsável(is) : Alice Mota Faleiro

Estatística Aplicada

1) Uma tela mostrando, em um período fornecido pelo usuário, a quantidade de vendas por dia, com os seguintes dados:
  a) Tabela de distribuição de frequência;
  b) Calcular e mostrar a média, moda, mediana, variância e desvio padrão;
  c) Visualizar um gráfico a partir da tabela.

2) Uma tela mostrando, em um período fornecido pelo usuário, o valor das vendas por dia, com os seguintes dados:
  a) Tabela de distribuição de frequência;
  b) Calcular e mostrar a média, moda, mediana, variância e desvio padrão;
  c) Visualizar um gráfico a partir da tabela.

3) Uma tela mostrando, em um período fornecido pelo usuário, os seguintes dados:
  a) Qual cliente compra mais vezes
  b) Qual cliente gasta mais.

4) Uma tela mostrando, em um período fornecido pelo usuário, uma tabela com a forma de pagamento e o cálculo da moda;

5) Fazer uma carta de controle do valor das vendas usando os limites a média e um desvio padrão e mostrar os valores fora dessa faixa através de um gráfico de dispersão.

Professor(es) responsável(is) : Fabiola de Carvalho Teixeira

Engenharia de Requisitos

Fazer o diagrama de Casos de Uso para o programa (com as descrições de cada Caso de Uso, conforme modelo definido em sala de aula);

Para cada Caso de Uso, utilizando a metodologia de Pontos de Caso de Uso, mensurar o esforço para o desenvolvimento.
